# SNS GOD 기능 안내

SNS GOD은 한 사람이 로컬 기기에서 사용하는 캐릭터 중심 메신저·SNS 시뮬레이션 앱입니다. 캐릭터 설정, 대화, 이미지, 기억과 자동 연락을 하나의 로컬 상태로 관리합니다.

## 연락과 대화

### 1:1·단체·랜덤 채팅

캐릭터와 개인 대화를 하거나 여러 캐릭터가 참여하는 단체방, 임시 랜덤 채팅을 사용할 수 있습니다.

- 사용: 하단 연락 화면에서 기존 방을 열거나 `+`로 새 방·단체방·캐릭터를 만듭니다.
- 설정: 캐릭터별 말투·관계·연락 성향과 방별 관계 메모를 사용할 수 있습니다.
- 특징: 캐릭터 Persona, 언어와 기억 정책을 채널 전체에서 공유합니다.
- 제한: AI 답장과 이미지 생성에는 선택한 provider 설정이 필요합니다.

### 로컬 답장 재시작 복구

1:1·랜덤 채팅에서 사용자 메시지를 보내면 원본 메시지와 예정 시각을 로컬 저장소에 먼저 기록합니다. 답장을 기다리거나 생성하는 도중 앱이 종료되어도 다음 실행에서 유효한 작업만 이어집니다.

- 사용: 별도 설정 없이 로컬 답장에 자동 적용됩니다.
- 기한 경과: 원래 답장 예정 시각이 지났다면 다시 긴 대기 연출을 하지 않고 과거 시각 기준으로 답장을 보정합니다.
- 중복 방지: 완료된 답장과 같은 작업 ID를 저장해, 생성 도중 종료되었더라도 이미 저장된 말풍선을 다시 만들지 않습니다.
- 자동 취소: 원본 메시지·방·캐릭터가 삭제되거나, 더 새 사용자 메시지를 보내거나, 방을 비활성화하거나, 전체 백업을 복원하면 이전 작업을 취소합니다.
- 제한: 먼저 연락 자동화 자체는 앱 프로세스가 종료된 동안 실행되지 않습니다.

### 먼저 연락과 자동화

앱을 실행해 둔 동안 캐릭터가 설정된 성향과 현재 상태에 맞춰 먼저 메시지, SNS 게시물 또는 전화를 만들 수 있습니다.

- 사용: 설정의 자동화에서 전체 기능과 채널별 옵션을 켭니다.
- 전제: Android 백그라운드 사용 시 알림 권한과 배터리 최적화 제외를 권장합니다.
- 하루 연락 예산: 같은 캐릭터의 개인 선톡, 단톡 자율 메시지, 전화 초대, 자동 SNS를 캐릭터 현지 날짜 기준의 한 예산으로 합산합니다. 방이나 채널을 바꿔도 총량을 우회하지 않습니다.
- 연락 순서: 캐릭터의 먼저 말 걸 확률, 현재 활동·에너지·휴대폰 상태와 사용자가 답하지 않은 횟수에 따라 개인톡·단톡·전화·SNS 우선순위가 달라집니다.
- 사용자 답장: 답장을 보내면 해당 캐릭터의 미응답 압력만 초기화되며, 이미 사용한 오늘의 연락 횟수는 복구되지 않습니다.
- 제한: 앱을 강제 종료하면 로컬 자동화가 중단됩니다. 다시 실행한 뒤부터 연락 예산과 저장된 상태를 기준으로 이어집니다.

## 알림

### 알림 목록과 target 이동

알림 행을 누르면 알림 종류에 맞는 실제 화면을 엽니다. 지원 target은 개인·단체·랜덤 채팅, SNS 게시물, SNS DM, 썸갓, 통화와 만남입니다.

- 사용: 연락 또는 ETC 화면의 알림 버튼을 누른 뒤 항목을 선택합니다.
- Android 진입: `snsgod://notifications`는 알림 목록을 열고, `snsgod://notification?id=<알림 ID>`는 저장된 알림을 찾아 같은 router로 처리합니다.
- 중복 방지: 수신 메시지와 SNS DM의 고유 ID를 로컬 사건 identity로 사용합니다. 같은 사건이 재시작·재처리·상태 병합으로 다시 들어와도 알림과 unread는 한 번만 반영됩니다.
- 묶음 표시: 짧은 시간 안에 여러 메시지가 오면 알림 한 항목으로 묶되, 표시 개수와 unread는 실제 새 메시지 수를 따릅니다.
- 읽음: 항목을 열거나 전체 `읽음`을 누르면 관련 알림, 채팅 unread 또는 SNS DM unread가 한 번에 정리됩니다. `정리`로 목록 항목도 지울 수 있습니다.
- 안전 처리: 삭제되었거나 불완전한 target은 엉뚱한 방을 열지 않고 알림 목록으로 돌아옵니다.

## 캐릭터와 기억

### 캐릭터 설정

이름, 프로필, 성격 prompt, 언어, 시간대, 날씨, 연락 성향, 이미지 외형 태그와 로어북을 캐릭터별로 관리합니다.

- 사용: 캐릭터 프로필 또는 설정의 캐릭터별 설정에서 편집합니다.
- 예시: 특정 캐릭터만 English를 사용하거나, 삽화 외형 태그로 장면 이미지의 외형 일관성을 높일 수 있습니다.
- 제한: `첫 메시지`는 새 캐릭터와 첫 방을 만들 때만 적용됩니다.

### 기억과 관계

장기 사실 기억, 별도 보관되는 추억 장면, 방 관계 메모와 대화 요약을 구분합니다.

- 사용: 캐릭터 설정의 프롬프트·메모리 탭과 방 설정을 사용합니다.
- 특징: 개인 기억과 단체방 공개 기억의 경계를 유지합니다.

## SNS와 발견

### 캐릭터 SNS·DM

Instagram 또는 X 형식의 게시물, 댓글과 DM을 캐릭터별로 생성하고 열람합니다.

- 사용: 하단 SNS 탭에서 플랫폼과 캐릭터를 선택합니다.
- 알림 연결: SNS 게시물·DM 알림을 누르면 해당 플랫폼과 대상 게시물 또는 thread를 엽니다.
- 설정: 캐릭터별 계정 분위기, 소재, 댓글·DM, 이미지 사용 여부를 조정합니다.

### 만남·데이트·썸갓

거리의 우연한 만남, 블라인드 데이트, 데이트 앱, 이상형 월드컵과 캐릭터 질문형 썸갓을 제공합니다.

- 사용: ETC 화면에서 원하는 기능을 선택합니다.
- 제한: 생성 기능은 설정된 AI provider와 이미지 provider 상태에 영향을 받습니다.

## 통화와 만남 이벤트

채팅에서 통화 초대를 받거나 직접 통화를 열 수 있고, 대화 맥락에서 만남 이벤트를 시작할 수 있습니다.

- 사용: 채팅의 통화 카드·프로필 통화 버튼·만남 카드를 누릅니다.
- 특징: 캐릭터 Persona와 현재 관계·기억을 통화와 만남 prompt에도 사용합니다.
- 상태 구분: 연결 전 거절·취소는 `cancelled`, 진행 중 뒤로가기는 `paused`, 정상 종료는 `finished`로 저장합니다. 취소된 세션은 다시 열리지 않습니다.
- 이어하기: 통화 대사·선택지·페이지와 만남 대사·선택지를 로컬에 저장합니다. 앱이 백그라운드로 가거나 종료된 뒤 돌아오면 진행 지점에서 재개하며, 채팅의 `통화 이어가기`·`이어서 만나기` 버튼으로도 다시 열 수 있습니다.
- 중복 방지: 정상 종료 결과, 통화 기록, 만남 결과 카드와 후속 메시지는 세션 ID 기준으로 한 번만 반영합니다.
- 삭제 정합성: 방이나 캐릭터를 삭제하면 연결된 통화·만남 세션과 활성 포인터도 함께 정리합니다.

## 이미지·앨범·백업

### 이미지와 앨범

채팅, SNS, 프로필, 만남에서 생성하거나 선택한 이미지를 통합 앨범에서 확인할 수 있습니다.

- 사용: ETC의 앨범을 열어 출처와 prompt를 확인합니다.
- 삭제: 연결 해제와 실제 미사용 파일 정리를 구분합니다. 공유 자산은 마지막 참조가 사라질 때만 정리 대상이 됩니다.

### 로컬 백업과 복원

- 사진 포함 전체 ZIP: 상태와 앱 관리 이미지를 함께 저장하고 복원합니다.
- 상태만 JSON: 사진을 제외한 캐릭터·대화·설정 상태를 저장합니다.
- 제한: API 키와 서버 인증 비밀값은 백업에 포함하지 않습니다.

## 설정과 provider

텍스트 AI provider, 이미지 provider, 사용자 프로필, 자동화, 알림·배터리 상태, 프롬프트와 로어북을 설정합니다.

- 로컬 원칙: 앱 데이터는 기본적으로 기기 내부에 저장됩니다.
- 네트워크: AI 생성이나 선택한 서버 기능을 사용할 때만 구성한 외부 provider로 요청합니다.
- 고급 기능: raw prompt와 provider 주소는 동작을 이해하는 경우에만 수정하는 편이 안전합니다.
