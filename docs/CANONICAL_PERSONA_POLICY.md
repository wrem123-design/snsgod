# Canonical Persona 정책

캐릭터의 정체성과 말투는 채널마다 별도 문자열로 다시 작성하지 않습니다. 모든 대화 채널은 `canonicalPersona.ts`가 만드는 동일한 block을 prompt compiler에 전달합니다.

## 공통 core

- 정체성: 캐릭터 ID, 이름, 핸들, 저장된 캐릭터 프로필
- 말투와 행동: 메시지 스타일과 반응·생각·리액션·톤 값
- 언어: 캐릭터별 언어가 있으면 전체 설정 언어보다 우선

개인톡, 단톡, 일반 선톡, 캘린더 선톡, 자동 단톡, 통화와 통화 종료 문구가 같은 core를 사용합니다. 채널 adapter는 JSON 형식, 메시지 수, 통화 turn 같은 출력 차이만 추가합니다.

## 관계와 기억 경계

- 개인톡·선톡·통화: 해당 캐릭터와 사용자의 private 사실, 방 관계, 공개 단톡에서 이어진 허용 맥락을 사용할 수 있습니다.
- 단톡·자동 단톡: 현재 방의 `group_public` 요약과 기억만 전달합니다.
- 단톡 prompt에는 `private_with_user` 원문이나 private hint를 넣지 않습니다. 필요한 개인 내용은 개인톡에서만 사용합니다.

각 memory block에는 visibility guard를 명시해 모델이 다른 채널의 숨은 기억을 추측하거나 공개하지 않게 합니다.
